# Checkpoint: 2026-05-27_1600

## Context Summary

Short maintenance session triggered by a live update failure. A user upgrading
from v2.9.0 received "Update failed: Download failed. Not Found" because the
release zip for v2.17.4 was never built and pushed when v2.17.4 shipped. The
session fixed the immediate bug and then hardened the release process to prevent
recurrence by adding a pre-push Git hook.

---

## Accomplishments

1. **v2.17.4 self-update bug fixed** (commit `229d6f1`)
   - Root cause: `releases/v2.17.4/rankmath-rest-bridge.zip` did not exist on
     GitHub. `update-manifest.json` pointed to it; WP fetched, got 404, surfaced
     "Download failed. Not Found".
   - Fix: ran `bin/build-zip.ps1` (all 4 structural checks passed -- 126 entries,
     234.6 KB), committed the zip, pushed.

2. **Pre-push hook added** (commit `2dd7221`)
   - New file: `hooks/pre-push` (tracked in git, bash script).
   - On every `git push`, hook reads version from `update-manifest.json`, checks
     if `releases/v{VERSION}/*.zip` exists, and if missing: runs
     `bin/build-zip.ps1`, auto-commits the zip, then proceeds with the push.
   - PowerShell detection: prefers `pwsh` (PS7+), falls back to `powershell`
     (PS5), hard-errors with clear manual instructions if neither found.
   - Push aborts if the build script exits non-zero or the zip is still absent
     after the build.

3. **`composer.json` updated** -- `post-install-cmd` and `post-update-cmd` now
   install both `hooks/pre-commit` and `hooks/pre-push` so a fresh
   `composer install` sets up the full hook suite.

4. **Release checklists updated** in `STARTUP_CONTEXT.md` and `.claude/CLAUDE.md`
   -- old 8-step flow trimmed to 6 steps; steps "build zip" and "commit zip" are
   now implicit in `git push`.
