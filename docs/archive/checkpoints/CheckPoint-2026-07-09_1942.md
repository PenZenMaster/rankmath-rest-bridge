## Accomplishments

- **Issue #3 fixed (v2.18.1, CLOSED)** -- POST /schema returned success but
  never persisted. True root cause (not the one suspected in the issue):
  v2.14.4 registered `_rrseo_schema_graph` and `_rrseo_change_log` via
  `register_post_meta()` with `type: string` + `sanitize_textarea_field`;
  WP applies registered sanitize callbacks on every `update_post_meta()` and
  string sanitizers return '' for arrays -- every schema write AND every
  audit-log append since v2.14.4 was silently discarded. Registrations
  removed; no data migration needed.
- **Issue #4 fixed (v2.18.1, CLOSED)** -- double canonical tag: plugin never
  unhooked WP core `rel_canonical` (wp_head:10). Emitter converted to named
  function `rr_emit_singular_canonical()`; calls
  `remove_action('wp_head','rel_canonical')` only when committed to emitting;
  all bail paths keep core's tag.
- **v2.18.1 deployed + QA passed** (user confirmed mid-session).
- **v3.0 Bite 2 SHIPPED (v2.19.0)** -- `includes/class-rrseo-actions.php`:

  audit rows via `rr_audit_log()`; both cache busts on every execute.
