## Accomplishments

### v2.3.x — Auto-update repair + manifest fix
- Diagnosed two silent bugs in `update-manifest.json`: missing `name` field caused
  PUC `validateMetadata()` to return `WP_Error` silently; `zip_url` field was ignored
  by PUC (property is `download_url`). Both bugs meant WordPress never showed update notices.
- Rewrote manifest to full PUC spec (`name`, `slug`, `download_url`, `sections`,
  `author`, `requires`, `requires_php`, `tested`).
- Built `releases/v2.3.1/rankmath-rest-bridge.zip` (177 KB, 118 entries).
- Flagged auto-update as a critical plugin feature in all project docs.

### v2.4.0 — Testing stack + pre-commit hook + migrate-legacy endpoint
- `composer.json` with PHPUnit 9.x, phpcs/wpcs; `vendor-dir: dev-vendor` to avoid PUC conflict.
- `phpcs.xml.dist` (WordPress-Core/Extra/Docs), `phpunit.xml.dist`.
- `tests/bootstrap.php` with WP stubs; 35 unit tests across SeoValidationTest,
  SchemaValidationTest, ManifestTest, TitleOutputTest.
- `hooks/pre-commit` committed; auto-installed via composer post-install-cmd.
- `POST /rankrocket-seo/v1/migrate-legacy` — batch-copies `rank_math_*` → `rr_seo_*` per post,
  dry_run support, audit log writes, batch max 20.

### v2.4.1 — WPCS compliance + title fix
- **1,804 pre-existing WPCS errors resolved** in two stages: phpcbf auto-fixed 1,766;
  38 manual fixes (Yoda conditions, short ternaries, docblocks, security escaping,
  in_array strict, parse_url → wp_parse_url, line lengths, @package tag).
- **Document title override bug fixed**: `add_filter('pre_get_document_title', ...)` was
  nested inside a `wp_head` callback at priority 1. WordPress core's `_wp_render_title_tag()`
  was registered first at the same priority, so it always fired before the plugin's callback —
  every `rr_seo_title` value was silently discarded. Fixed by registering the filter at
  plugin-load time (priority 20).
- Pre-commit hook set to blocking (`EXIT_CODE=1`); phpcs now enforces 0 errors on commit.
- phpcs.xml.dist extended to scan `includes/` in addition to main file.

### v2.5.0 — WordPress admin panel + Edit Post/Page meta box
- `includes/class-rrseo-admin.php` — RRSEO_Admin class; 6-page admin menu:
  Overview, Posts & Pages, Image ALT, Snippets, llms.txt, Sitemap Preview.
- `includes/class-rrseo-metabox.php` — read-only sidebar meta box on Edit Post/Page;
  no save_post hook; JS fetches SEO meta, schema type, last 3 audit entries post-load.
- `includes/admin.js` — 6-page JS router using wp.apiFetch; all data from existing REST endpoints.
- `includes/metabox.js` — parallel Promise.all fetch of /get/{id}, /schema/{id}, /log/{id};
  character-count badges (green/amber/red) matching PHP validation thresholds.
- `includes/admin.css` — shared layout CSS, no external framework.
- Loaded conditionally via `if ( is_admin() )` — zero front-end cost.

### v2.6.0 — Styled sitemaps + per-type sub-sitemaps
- `includes/sitemap.xsl` — XSL stylesheet served by PHP at `/rmb-sitemap.xsl`;
  dark navy header, clean white content panel, alternating rows, priority badges.
  PHP injects `RRSEO_SITE_NAME`, `RRSEO_INDEX_URL`, `RRSEO_VERSION` tokens.
  Handles both `<sitemapindex>` (sub-sitemap table) and `<urlset>` (URL table) views.
- Per-type sub-sitemaps: `/sitemap_index.xml` now lists `/rmb-sitemap-posts.xml` and
  `/rmb-sitemap-pages.xml` (evaluated against the reference site trevoraspiranti.com pattern).
  The old single-entry index was semantically equivalent to having no index at all.
- Accurate UTC `lastmod`: switched from `post_modified` (local, mislabeled +00:00) to
  `post_modified_gmt` throughout.
- Legacy combined `/rmb-sitemap.xml` kept for backward compatibility.
- New route handler: `rmb_serve_sitemap_xsl()`, `rmb_serve_sitemap_type()`.

### v2.7.0 — GET/POST /robots-txt endpoint
- `GET /rankrocket-seo/v1/robots-txt` — returns current custom content, source flag
  (`custom` | `wordpress_default`), `physical_file_exists` bool, warning string.
- `POST /rankrocket-seo/v1/robots-txt { "content": "..." }` — writes `rrseo_robots_txt`
  option; `content: ""` clears override and restores WP default.
- `robots_txt` filter registered at plugin-load (priority 99) via `rr_robots_txt_output()`.
  Warns via API when a physical `robots.txt` exists at ABSPATH (web server bypasses WP filter).
- **Release zip bug discovered and fixed**: v2.7.0 zip was built with a flawed PowerShell
  command that flattened PUC subdirectories. Caused fatal error on plugin activation post-update
  (`Puc/v5p5/Autoloader.php` not found). Rebuilt correctly.
- `bin/build-zip.ps1` — standardized release builder with 4 structural verification checks;
  exits non-zero on failure. Prevents recurrence of the flat-vendor bug.
