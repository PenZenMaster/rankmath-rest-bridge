### v2.9.2 — Three P1 gap fixes

**Fix 1: `rankmath-bridge/v1` legacy namespace alias**
- `rr_legacy_namespace_proxy()` hooked on `rest_pre_dispatch` (priority 10)
- Intercepts `/rankmath-bridge/v1/...`; creates a new `WP_REST_Request` for the same
  method/body routed to `/rankrocket-seo/v1/...`; dispatches internally
- Appends `_deprecated { deprecated_namespace, preferred_namespace, message }` to every
  aliased response
- No duplicate route registrations — zero boilerplate

**Fix 2: `register_post_meta` for `_rrseo_llms_section`**
- `init` hook calls `register_post_meta()` for `META_LLMS_SECTION` on every post type
  in `rrseo_metabox_post_types` filter (default: post, page)
- `show_in_rest: true` → field now visible in `/wp/v2/posts` REST responses
- `sanitize_callback: 'sanitize_text_field'` → WP sanitizes on save
- `auth_callback` → restricts REST writes to `manage_options`

**Fix 3: First-paragraph description fallback bug**
- Root cause: `rr_normalize_description_text()` collapses `\n\n` to spaces BEFORE the
  paragraph-split regex ran. `preg_split('/\s{2,}/', $normalized_text)` always returned
  one element → every content-only post silently fell through to title + `thin_description`
- Fix: split raw content on `\n\n`, `</p>`, `<br><br>` FIRST, then normalize each
  extracted paragraph individually
- 5 new tests covering the full fallback priority chain and both content formats
