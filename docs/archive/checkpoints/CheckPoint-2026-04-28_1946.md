## Accomplishments

1. **Mental model rename + native meta keys (v2.2.0)**
   - Plugin Name header updated to "RankRocket SEO Control Layer"
   - REST namespace changed: `rankmath-bridge/v1` -> `rankrocket-seo/v1`
   - `RR_SEO_META_KEYS` and `RR_SEO_LEGACY_META_KEYS` constants defined
   - `rr_get_seo_meta()` helper: native `rr_seo_*` first, `rank_math_*` fallback
   - All internal reads (wp_head, sitemap, llms.txt) use the helper
   - All writes target `rr_seo_*` only
   - Option keys and PUC slug preserved to avoid data loss on existing installs

2. **Schema model + preview endpoint + validation layer + audit log (v2.3.0)**
   - `GET/POST /rankrocket-seo/v1/schema/{post_id}` — stored as `_rrseo_schema_graph`
   - JSON-LD injected via `wp_head` on singular posts
   - `POST /rankrocket-seo/v1/preview-update` — before/after diff, no DB write
   - `rr_validate_seo_fields()`: title 30-60 warn/120 hard, desc 50-160 warn/320 hard,
     og_image URL check, robots token allowlist, post type allowlist, batch max 20
   - `rr_validate_schema()`: JSON parse, @context required, @type in 17-type allowlist
   - Both allowlists extensible via `apply_filters`
   - `rr_audit_log()`: appends to `_rrseo_change_log` postmeta, capped at 100/post
   - `GET /rankrocket-seo/v1/log/{post_id}` — returns log most-recent-first
   - `/update` and `/meta/bulk-update` now validate and audit every write
   - Bulk update: per-item validation (bad items fail individually, batch continues)

3. **replace-all hardened (v2.3.1)**
   - `RR_REPLACE_ALL_CAP = 'rrseo_replace_all_snippets'` constant
   - `init` hook grants cap to administrator role on first load (idempotent)
   - Route permission callback checks custom cap instead of broad `manage_options`
   - Success response includes `deprecated` field; route annotated `@deprecated`
   - Target removal: v3.0.0

4. **RRSEO session management scaffolded**
   - `.claude/claude.md` — project playbook with start/checkpoint/shutdown commands
   - `.claude/settings.json` — project-level permission allowlist
   - `docs/STARTUP_CONTEXT.md` — lightweight session primer
   - `docs/projectStatus.md` — full version history and backlog
   - `docs/archive/checkpoints/` — checkpoint archive
